## Architecture

IoProvider trait abstracts all terminal input operations:
- read_line(prompt) → Result<String, IoError>
- read_key() → Result<String, IoError>
- read_password(prompt) → Result<String, IoError>

Implementations:
- StdIoProvider: reads from real stdin, returns NoInputAvailable on EOF
- MockIoProvider: pre-loaded VecDeque responses for deterministic tests

ExecContext carries Arc<dyn IoProvider> through the execution chain.
run_script_with_provider(source, debug, provider) is the new entry point.
run_script(source, debug) is preserved for backward compatibility (uses StdIoProvider).
