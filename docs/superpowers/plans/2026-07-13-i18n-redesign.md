# i18n Infrastructure Redesign — Phase 0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the hand-written `pub fn` per-message i18n system with a key-value table backed by JSON resource files, adding fallback, plugin registration, and coverage-gate testing — as the prerequisite for the plugin ecosystem.

**Architecture:** Two JSON resource files (`locales/en.json`, `locales/zh.json`) are embedded at compile time via `include_str!`. A `MessageStore` struct holds parsed `HashMap<String, String>` tables for en/zh plus a `plugins` map for plugin-registered messages. The core `t(lang, key, args)` function looks up a key, interpolates `{placeholder}` args, and falls back zh→en with a one-time warning. Existing `pub fn` messages become thin wrappers calling `t()` so call sites are untouched.

**Tech Stack:** Rust 2021, `serde_json` (already a dependency), `std::collections::HashMap`. No new dependencies.

**Spec:** [docs/superpowers/specs/2026-07-13-plugin-ecosystem-design.md](file:///workspace/docs/superpowers/specs/2026-07-13-plugin-ecosystem-design.md) §12

## File Structure

| File | Responsibility |
|---|---|
| `locales/en.json` (create) | All English message templates, keyed by `namespace.action` |
| `locales/zh.json` (create) | All Chinese message templates, same keys |
| `src/i18n.rs` (rewrite) | `MessageStore`, `t()`, `register_plugin()`, `all_keys()`, `missing_translations()`, thin-wrapper `pub fn`s |
| `tests/i18n_tests.rs` (create) | Coverage gate, bilingual assertions, interpolation, fallback, plugin registration |

**Lint note:** Project denies `unwrap_used`, `panic`, `indexing_slicing`, `todo`, `unimplemented`. All code must use `?`, `expect` is also denied under pedantic — use `unwrap_or_default` or proper `Result` propagation. HashMap access must use `.get()` not `[]`.

- [ ] **Step 3: Rewrite `src/i18n.rs` — keep Lang enum, replace message functions with MessageStore + t()**

The top of the file (Lang enum, from_name, name, Default) stays unchanged. Replace everything from `// ── Debug prefix helper ──` down to (but not including) `// ── Tests ──` with:

```rust
// ── Message store (key-value table backed by JSON resources) ───────

use std::collections::HashMap;
use std::sync::OnceLock;

/// Plugin-registered message tables.
#[derive(Default)]
pub struct PluginMessages {
    en: HashMap<String, String>,
    zh: HashMap<String, String>,
}

/// Central message store. Holds dyyl's own en/zh tables (compiled in)
/// plus plugin-registered tables.
pub struct MessageStore {
    en: HashMap<String, String>,
    zh: HashMap<String, String>,
    plugins: HashMap<String, PluginMessages>,
}

static STORE: OnceLock<MessageStore> = OnceLock::new();

const EN_JSON: &str = include_str!("../locales/en.json");
const ZH_JSON: &str = include_str!("../locales/zh.json");

fn parse_json(json: &str) -> HashMap<String, String> {
    serde_json::from_str(json).unwrap_or_default()
}

fn init_store() -> MessageStore {
    MessageStore {
        en: parse_json(EN_JSON),
        zh: parse_json(ZH_JSON),
        plugins: HashMap::new(),
    }
}

fn store() -> &'static MessageStore {
    STORE.get_or_init(init_store)
}

/// Look up a message by key, interpolate `{placeholder}` args, and return
/// the rendered string. Falls back zh→en if the zh key is missing (emits a
/// stderr warning). If neither language has the key, returns the key name
/// itself with a warning.
#[must_use]
pub fn t(lang: Lang, key: &str, args: &[(&str, &str)]) -> String {
    let s = store();
    // Determine which plugin table to check first (if key starts with "<plugin>.")
    let plugin_name = key.split('.').next().unwrap_or(key);
    let template = s.lookup_template(lang, key, plugin_name);
    interpolate(&template, args)
}

impl MessageStore {
    fn lookup_template(&self, lang: Lang, key: &str, plugin_name: &str) -> String {
        // 1. Plugin table (if this plugin is registered and key starts with plugin name)
        if key.starts_with(&format!("{plugin_name}.")) {
            if let Some(pm) = self.plugins.get(plugin_name) {
                if let Some(tpl) = match lang {
                    Lang::En => pm.en.get(key),
                    Lang::Zh => pm.zh.get(key).or_else(|| pm.en.get(key)),
                } {
                    return tpl.clone();
                }
            }
        }
        // 2. dyyl main table
        let main_tpl = match lang {
            Lang::En => self.en.get(key),
            Lang::Zh => self.zh.get(key).or_else(|| {
                eprintln!("i18n warning: zh translation missing for '{key}', falling back to en");
                self.en.get(key)
            }),
        };
        match main_tpl {
            Some(tpl) => tpl.clone(),
            None => {
                eprintln!("i18n warning: no translation found for '{key}'");
                key.to_string()
            }
        }
    }
}

/// Replace `{placeholder}` occurrences in template with provided args.
/// Unknown placeholders are left as-is.
fn interpolate(template: &str, args: &[(&str, &str)]) -> String {
    let mut result = template.to_string();
    for (name, value) in args {
        let placeholder = format!("{{{name}}}");
        result = result.replace(&placeholder, value);
    }
    result
}
```

**Note on `plugins` field mutability:** In Task 2 the `plugins` field is a plain `HashMap` (read-only via `&self`). Task 3 changes it to `Mutex<HashMap<...>>` to allow `register_plugin()` to insert at runtime. The `lookup_template` in Task 2 reads `self.plugins.get(...)` which works on `&self`; Task 3 updates this to `self.plugins.lock()...`.

**Note on `unwrap_or_default`:** `parse_json` uses `unwrap_or_default()` (returns empty HashMap on parse failure) — this is allowed under the `unwrap_used` deny lint since `unwrap_or_default` is not `unwrap`. An empty table will cause `missing_translations` tests to fail, surfacing the problem in CI.

    fn lookup_template(&self, lang: Lang, key: &str, plugin_name: &str) -> String {
        if key.starts_with(&format!("{plugin_name}.")) {
            let plugins = self.plugins.lock().expect("plugins mutex poisoned");
            if let Some(pm) = plugins.get(plugin_name) {
                if let Some(tpl) = match lang {
                    Lang::En => pm.en.get(key),
                    Lang::Zh => pm.zh.get(key).or_else(|| pm.en.get(key)),
                } {
                    return tpl.clone();
                }
            }
        }
        let main_tpl = match lang {
            Lang::En => self.en.get(key),
            Lang::Zh => self.zh.get(key).or_else(|| {
                eprintln!("i18n warning: zh translation missing for '{key}', falling back to en");
                self.en.get(key)
            }),
        };
        match main_tpl {
            Some(tpl) => tpl.clone(),
            None => {
                eprintln!("i18n warning: no translation found for '{key}'");
                key.to_string()
            }
        }
    }
}

use std::collections::HashMap;

#[test]
fn t_fallback_zh_to_en_emits_correct_text() {
    // Register a plugin with en only, no zh for this key
    let mut en = HashMap::new();
    en.insert("fallbacktest.hello".to_string(), "hello world".to_string());
    register_plugin("fallbacktest", en, HashMap::new());

    // zh should fall back to en text
    let msg = t(Lang::Zh, "fallbacktest.hello", &[]);
    assert_eq!(msg, "hello world");
}

#[test]
fn t_plugin_key_resolves_from_plugin_table() {
    let mut en = HashMap::new();
    en.insert("regtest.greeting".to_string(), "hi from plugin".to_string());
    let mut zh = HashMap::new();
    zh.insert("regtest.greeting".to_string(), "插件问候".to_string());
    register_plugin("regtest", en, zh);

    assert_eq!(t(Lang::En, "regtest.greeting", &[]), "hi from plugin");
    assert_eq!(t(Lang::Zh, "regtest.greeting", &[]), "插件问候");
}
